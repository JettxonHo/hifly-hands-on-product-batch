# 飞影批量生产新人手册设计

## 目标

提供一份可离线打开、也可打印的 HTML 使用手册，面向两类读者：

- 客户和合作方：理解需要提供什么素材、如何填写信息、如何验收交付物。
- 内部运营人员：按当前项目的网页自动化流程完成环境准备、批量生产、下载、质检和失败处理。

手册不改变自动化脚本、飞影账号或生产逻辑。

## 交付形式

- 文件：`docs/新人培训使用手册.html`
- 单一 HTML 文件，内嵌样式和少量原生 JavaScript；不依赖网络、框架或构建步骤。
- `README.md` 增加手册入口。
- 页面适合桌面阅读和 A4 打印，手机端可纵向阅读。

## 信息架构

1. 概览：用途、角色分工、开始前检查、页面目录。
2. 客户协作区：素材清单、商品图片要求、商品信息字段、人物/背景素材建议、交付验收标准。
3. 运营执行区：环境准备、登录、填写表格、人物素材池、校验、批量运行、确认弹窗、下载与归档。
4. 常见问题：登录失效、上传失败、页面按钮变化、任务失败、姿势/背景重复、下载失败。
5. 附录：关键命令和项目目录说明。

## 关键内容约束

- 如实说明“手里有货”模式未在官方 API 中开放，因此采用网页自动化。
- 明确当前页面的必要顺序：上传人物和商品图 → 弹窗内立即生成 → 完成后点击确认 → 外层立即生成 → 最新作品下载。
- 强调缺少人物图时的选择优先级：商品表明确人物图、同品类人物池、默认人物池、飞影推荐人物。
- 明确当前页面没有姿势参数；姿势、场景与景别由人物素材或飞影推荐模板主导。
- 不在手册中放账号、密码、真实客户素材或本地敏感配置。

## 页面交互与可用性

- 顶部锚点导航跳转到客户区、运营区、问题排查和附录。
- FAQ 使用原生 `details` 元素展开，不依赖脚本。
- 常用命令旁提供复制按钮；JavaScript 不可用时命令仍完整可见。
- 提供打印按钮，打印时隐藏导航和交互控件，保留正文。
- 色彩用于区分“客户”“运营”“关键提醒”“风险”，但不单独依赖颜色传达信息。

## 验证

- 在浏览器直接打开 HTML，检查目录跳转、FAQ 展开、复制与打印按钮。
- 以桌面和窄屏宽度检查文本不溢出、步骤顺序清晰、打印样式可读。
- 复核命令、路径和流程与 `README.md`、`docs/SOP.md`、`docs/ENVIRONMENT.md` 一致。
