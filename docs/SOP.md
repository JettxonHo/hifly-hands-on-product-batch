# 飞影「手里有货」批量生产 SOP

## 1. 输入标准

客户至少提供：

- 商品图片：主体清晰、无严重遮挡，优先白底图或干净场景图。
- 产品名称：不超过 20 个字更适合口播和字幕。
- 核心卖点：建议 3 条以内，用分号分隔。

运营补充：

- `sku`：唯一编号。
- `script`：可为空；为空时后续由文案生成模块补齐。
- `avatar`、`voice`：为空时使用配置默认值。
- `duration_seconds`：建议 15-25 秒。
- `status`：新任务填 `pending`。

## 2. 批量生产流程

1. 把商品图片放入 `products/images/`，按 SKU 命名。
2. 在 `products/products.csv` 填入商品信息。
3. 运行 `npm run validate` 检查字段和图片路径。
4. 首次使用运行 `npm run login`，人工登录飞影工作台。
5. 运行 `npm run run` 批量进入 `https://hifly.cc/goods` 的「手里有货」模式生成视频。
6. 检查 `downloads/`、`logs/`、`screenshots/`。
7. 失败任务修正后改回 `pending` 或保留失败状态由脚本重试。

当前飞影页面必须经过弹窗流程：

1. 外层点击「上传人物+产品图」。
2. 弹窗内上传人物图和商品图；如果商品表没有 `person_image_path`，脚本会选择推荐人物图。
3. 弹窗内点击「立即生成」。
4. 生成完成后点击「确认」。
5. 回到外层点击「立即生成」生成视频。
6. 在「最新作品」卡片右下角点击下载图标。

## 3. 质检标准

每条视频交付前检查：

- 文件可播放，非 0 字节。
- 竖屏比例，时长符合客户要求。
- 商品外观没有明显变形，包装文字没有严重错乱。
- 数字人动作和口型自然，无明显卡顿。
- 卖点表达不夸大，不触发平台违禁词。
- 字幕、产品名、CTA 没有错字。

## 4. 风险与兜底

- 登录失效或验证码：人工登录后重跑。
- 页面按钮或输入框文案变化：更新 `config.local.json` 的 `hiflyUi` 文案。
- 上传失败：检查商品图片格式、大小和路径。
- 生成失败：记录错误截图，降低批量并重试。
- 成片质量差：更换商品图或降低对“真实手持”的承诺，进入人工复核。
