# 飞影电商产品数字人种草视频 V1 工作流

## 项目目标

用飞影批量生产电商产品数字人手持商品种草视频。V1 只做单版本标准化交付：每个商品 1 条视频，不做多达人、多脚本、多平台 A/B 版本。

## 样片风格

- 竖屏 9:16。
- 左右分屏：左侧数字人口播，右侧手持商品或商品近景。
- 背景真实、光线自然，尽量让左右画面处在同一类场景中。
- 口播轻种草，不做硬广式堆卖点。
- 商品第一秒露出，卖点表达短句化。

## 角色分工

| 角色 | 负责内容 |
| --- | --- |
| 客户 | 提供商品图片、产品名称、核心卖点，确认样片风格。 |
| 运营 | 整理商品表、检查素材、补充品类、发起批量任务。 |
| 文案 | 将产品名和卖点改写为 15-25 秒口播脚本。 |
| 生成执行 | 在飞影完成上传、生成、下载和失败重试。 |
| 质检 | 检查商品还原、口播、字幕、违规表达和成片命名。 |

## 输入资料

客户最少提供三项：

1. 商品图片：主体完整，图片清晰，优先白底或干净场景图。
2. 产品名称：用于口播、字幕和文件命名。
3. 核心卖点：建议 2-3 条，用短句表达。

运营补充：

1. SKU：唯一商品编号。
2. 商品品类：用于匹配人物池和场景，例如 `fresh_food`、`beauty`、`snacks`。
3. 视频时长：默认 20 秒。
4. 状态：新任务填 `pending`。

## 标准生产链路

1. 素材接收：收齐商品图、产品名、核心卖点。
2. 表格整理：录入 `products/products.csv` 或 `products/商品信息表.xlsx`。
3. 素材命名：商品图放入 `products/images/`，按 SKU 命名。
4. 脚本生成：按固定 5 段式生成 80-120 字口播。
5. 提示词生成：按左右分屏模板生成飞影画面提示词。
6. 校验：运行 `npm run validate`，确认字段和图片路径正确。
7. 标准资产：运行 `npm run prepare-standard`，生成脚本、提示词和质检表。
8. 人工抽查：先看 `outputs/standard-video-assets/`，确认文案没有违规或跑偏。
9. 登录：首次或登录失效时运行 `npm run login`。
10. 执行：运行 `npm run run`，逐条进入飞影「手里有货」生成。
11. 下载：成片进入 `downloads/` 或批次产物目录。
12. 质检：按质检表检查，通过后进入交付目录。

## 单条视频脚本结构

| 段落 | 时长 | 内容 |
| --- | --- | --- |
| 钩子 | 0-3 秒 | 点出痛点、场景或直接利益点。 |
| 产品 | 3-7 秒 | 说清产品名称和适用场景。 |
| 卖点 | 7-14 秒 | 讲 2-3 个核心卖点。 |
| 体验 | 14-18 秒 | 用生活化语言转成使用感。 |
| CTA | 18-25 秒 | 轻引导收藏、进店或下单。 |

## 产能建议

- 样片校准：3 条，覆盖不同品类。
- 小批量试产：10-20 条，观察失败率。
- 稳定生产：20-50 条一批，批次内不换模板。

## 验收标准

- 每个 SKU 只有 1 条标准视频。
- 视频可播放，竖屏比例正确，时长 15-25 秒。
- 商品不明显变形，包装文字不严重错乱。
- 数字人口型和动作自然。
- 卖点与客户提供信息一致，无编造功效。
- 字幕、产品名称和 CTA 无错字。
- 文件按 `sku_product_name_standard.mp4` 命名。

## 后续升级方向

V1 稳定后再升级：

- 每商品多达人版本。
- 每商品多脚本版本。
- 不同平台投放比例和字幕样式。
- 自动生成脚本、提示词、质检报告。
- 基于转化数据的批量复刻模板。
