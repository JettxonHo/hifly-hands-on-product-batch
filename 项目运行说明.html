飞影批量种草视频项目运行说明

适用于“每个商品只出 1 条标准视频”的 V1 工作流：商品表输入 → 校验 → 生成脚本/提示词/QC 表 → 飞影批量生成 → 下载质检。

项目目录：/Users/ketchup/Documents/Product Recommendation clip

1. 首次安装

打开终端，先进入项目目录：

cd "/Users/ketchup/Documents/Product Recommendation clip"

安装依赖和 Playwright 浏览器：

npm install
npx playwright install chromium

复制本地配置文件：

cp config.example.json config.local.json

2. 首次登录飞影

首次使用或登录失效时运行：

npm run login

系统会打开浏览器。人工完成飞影登录，并确认能进入飞影工作台后，回到终端按提示继续。

3. 准备商品资料

商品信息可以填在以下任意一种表里：

  • products/products.csv
  • products/商品信息表.xlsx

商品图片统一放入：

products/images/

每个商品至少需要这些字段：

字段 说明 示例
sku 唯一商品编号，用于命名和追踪任务。 SKU001
product_name 产品名称，用于口播、字幕和文件名。 山野小青菜
selling_points 核心卖点，建议 2-3 条，用中文分号分隔。 根系完整；叶片厚实；清炒清甜
category 商品品类，用于匹配人物池和场景。 fresh_food
image_path 商品图片路径。 products/images/SKU001.png
status 新任务填 pending。 pending

4. 标准运行流程

准备好商品表和图片后，按顺序运行：

npm run validate
npm run prepare-standard
npm run run
命令 作用
npm run validate 检查商品表字段、图片路径和人物素材配置。
npm run prepare-standard 为每个待处理商品生成口播脚本、飞影提示词和质检表骨架。
npm run run 启动自动化浏览器，进入飞影“手里有货”页面批量生成并下载视频。

5. 本地网页工作台

如果想用本地网页界面录入或导入商品，运行：

npm run gui

终端会打印类似下面的地址：

Local workbench: http://127.0.0.1:4317

打开这个地址即可使用本地工作台。如果端口被占用，系统会自动换到下一个可用端口。

6. 产物位置

目录 内容
outputs/standard-video-assets/scripts/ 每个商品的标准口播脚本。
outputs/standard-video-assets/prompts/ 每个商品的飞影左右分屏提示词。
outputs/standard-video-assets/qc/qc_report.csv 质检表骨架。
downloads/ 飞影下载回来的视频文件。
logs/ 运行日志。
screenshots/ 失败截图和调试截图。

7. 常见问题

登录失效或出现验证码

重新运行：

npm run login

商品校验失败

先看终端错误提示，常见原因是商品图片路径写错、必填字段为空、状态没有填 pending。

飞影页面按钮找不到

飞影页面文案或结构可能变了，需要检查 config.local.json 里的 hiflyUi 配置。

成片质量不稳定

优先检查商品图是否清晰、主体是否完整；包装类商品建议用干净白底图或透明底抠图。

V1 口径：每个商品只出 1 条标准视频。多达人、多脚本、多版本投放测试后续再升级。
